    Magic Variables

    These are variables that are always there, without you having to import them.

    The delay between the updating variables is always 2s.

    Updating magic variables

    EWW_TEMPS

    Heat of the components in Celcius

    Example: {{(CPU_TEMPS.core_1 + CPU_TEMPS.core_2) / 2}}

    EWW_RAM

    The current RAM + Swap usage

    EWW_DISK

    Information on on all mounted partitions (Might report inaccurately on some filesystems, like btrfs)

    Example: {{EWW_DISK["/"]}}

    EWW_BATTERY

    Battery capacity in procent of the main battery

    EWW_CPU_USAGE

    Average CPU usage (all cores) since the last update (No MacOS support)

    EWW_NET

    Bytes up/down on all interfaces
